Exploring Python decorators

posted in Technology on December 22 2015Comments

Decorator trong Python được sử dụng khá nhiều, tuy nhiên nó cũng hơi khó hiểu một chút. Decorator cho chúng ta một cú pháp đơn giản để gọi các hàm bậc cao (higher-order functions). Về mặt lý thuyết, một decorator là một hàm nhận tham số đầu vào là một hàm khác và mở rộng tính năng cho hàm đó mà không thay đổi nội dung của nó. Tuy nhiên, decorator không chỉ có vậy, nó thực sự còn làm nhiều việc khác nữa. Chúng ta sẽ dần dần khám phá trong bài viết này.

Trong bài viết này, tôi sẽ giới thiệu một cách ngắn gọn và dễ hiểu về decorator, cách sử dụng chúng và cách tự viết một decorator cho riêng mình.

Từ "decorator" được sử dụng trong Python có một chút biến đổi. Bạn hãy cẩn thận vì nó rất dễ gây nhầm lẫn với một design pattern cũng tên là decorator. Thực ra, chúng ta có thể sử dụng decorator trong Python để cài đặt decorator pattern. Nhưng có rất nhiều hạn chế trong cách làm này. Theo tôi thấy thì decorator của Python có nhiều nét giống với macro hơn.

Cơ sở lý thuyết

Trước khi bạn có thể hiểu các decorator, thì bạn cần hiểu một số khái niệm trước. Trong phần này, chúng ta sẽ tìm hiểu những khái niệm đó.

Các hàm làm việc như thế nào

Đương nhiên, các hàm sẽ trả về kết quả dựa vào tham số của nó:

def foo(bar):
    return bar ** 2

print(foo(2) == 4)

Hàm trên rất dể hiểu phải không nào?

Các đối tượng first-class

Trong Python, các hàm là những đối tượng first-class. Điều đó có nghĩa là, các hàm này có thể được sử dụng như những tham số, giống như các giá trị thông thường khác (ví dụ string, int, float).

def foo(bar):
    return bar + 1

print(foo)
print(foo(2))
print(type(foo))

def call_foo_with_arg(foo, arg):
    return foo(arg)

print(call_foo_with_arg(foo, 2))

Các hàm lồng nhau

Vì các hàm là các đối tượng first-class trong Python, nên bạn có thể định nghĩa một hàm bên trong một hàm khác. Những hàm như vậy gọi là các hàm lồng nhau.

def parent():
    print("This is parent() function")

    def first_child():
        return "This is first_child() function"

    def second_child():
        return "This is second_child() function"

    print(first_child())
    print(second_child())

Điều gì sẽ xảy ra khi bạn gọi làm parent()? Tôi nghĩ nó không đến nỗi quá khó hiểu...

This is parent() function
This is first_child() function
This is second_child() function

Bây giờ, nếu bạn gọi làm first_child(), bạn sẽ nhận được thông báo lỗi:

NameError: name 'first_child' is not defined

Bài học rút ra

Bất cứ khi nào chúng ta gọi hàm parent(), hai hàm con là first_child() và second_child() cũng sẽ được gọi. Nhưng vì 2 hàm con này được bao bởi hàm parent() nên chúng không thể gọi từ bên ngoài hàm parent() được.

Một hàm trả kết quả là một hàm khác

Python cho phép bạn viết một hàm mà hàm đó trả kết quả về là một hàm khác. Hãy thay đổi code ở ví dụ trên một chút như sau:

def parent(num):

    def first_child():
        return "This is first_child() function"

    def second_child():
        return "This is second_child() function"

    if num == 10:
        return first_child
    else:
        return second_child

foo = parent(10)
bar = parent(20)

print(foo)
print(bar)

print(foo())
print(bar())

Output của 2 lệnh print đầu tiên sẽ là:

<function parent.<locals>.first_child at 0x7f3a6aa11b70>
<function parent.<locals>.second_child at 0x7f3a6aa11a60>

Đây đơn giản chỉ là thông tin cho chúng ta biết rằng foo đã được trỏ tới hàm first_child() và bar đã được trỏ tới hàm second_child().

Output của 2 hàm print tiếp theo đã chứng minh điều đó:

This is first_child() function
This is second_child() function

Bạn có để ý rằng, ở ví dụ trước, chúng ta gọi 2 hàm con từ trong hàm bố mẹ bằng cách gọi rất đầy đủ first_child(). Trong khi ở ví dụ này, khi trả về các hàm con, chúng ta trả về chính hàm đó chứ không phải kết quả của nó, nên chúng ta không dùng dấu ngoặc, ví dụ first_child. Bằng cách gọi này, những hàm này có thể được sử dụng trong tương lai.

Python decorator

Bây giờ, chúng ta đã sẵn sàng tìm hiểu về decorator trong Python. Hãy xem qua 2 ví dụ sau:

Ví dụ 1:

def my_decorator(some_function):

    def wrapper():
        print("Something happens before some_function")
        some_function()
        print("Something happens after some_function")

    return wrapper

def just_some_function():
    print("Hello, world!")

just_some_function = my_decorator(just_some_function)
just_some_function()

Bạn có thể đoán được output của đoạn code trên hay không?

Something happens before some_function
Hello, world!
Something happens after some_function

Để hiểu được tại sao lại có output như trên, bạn có thể xem lại các ví dụ ở phần trước. Chúng ta chỉ đơn giản là áp dụng cơ sở lý thuyết mà chúng ta đã biết. Rất đơn giản, decorator cũng làm việc tương tự như vậy, đóng gói 1 hàm, mở rộng hành vi của hàm đó.

Hãy thêm một chút xử lý nữa.

Ví dụ 2:

def my_decorator(some_function):

    def wrapper():
        num = 10
        if num == 10:
            print("YES!!")
        else:
            print("NO!1")
        some_function()
        print("Something happens after some_function")

    return wrapper

def just_some_function():
    print("Hello, world!")

just_some_function = my_decorator(just_some_function)
just_some_function()

Output của đoạn code trên sẽ là:

YES!!
Hello, world!
Something happens after some_function

Cú pháp decorator của Python

Python cho phép bạn gọi decorator rất dễ dàng bằng cách sử dụng ký hiệu @ (được gọi là "pie syntax")

Tạo một module cho decorator

def my_decorator(some_function):

    def wrapper():
        num = 10
        if num == 10:
            print("YES!!")
        else:
            print("NO!!")
        some_function()
        print("Something happens after some_function")

    return wrapper

if __name__ == "__main__":
  my_decorator()

Giả sử chúng ta lưu đoạn code trên vào 1 file tên là decorator.py. Dưới đây là cách chúng ta gọi một hàm với decorator:

from decorator import my_decorator

@my_decorator
def just_some_function():
  print("Hello, world!")

just_some_function()

Khi bạn chạy đoạn code trên. Bạn sẽ nhận được kết quả giống hệt kết quả ở ví dụ trước.

YES!!
Hello, world!
Something happens after some_function

Vậy, cú pháp @my_decorator là cách gọi ngắn gọn của just_some_function = my_decorator(just_some_function).

Ví dụ thực tế

Dưới đây là một số ví dụ decorator được dùng trong thực tế.

Decorator cho hàm không có tham số

Ví dụ này rất dễ hiểu, nó hoàn toàn áp dụng cơ sở lý thuyết ở trên chúng ta đã phân tích.

import time

def timing_function(some_function):
    """
    Outputs the time a function takes
    to execute.
    """

    def wrapper():
        t1 = time.time()
        some_function()
        t2 = time.time()
        return "Time it took to run the function: " + str((t2-t1)) + "\n"
    return wrapper

@timing_function
def my_function():
    num_list = []
    for x in (range(0,10000)):
        num_list.append(x)
    print("\nSum of all the numbers: " + str((sum(num_list))))

print(my_function())

Decorator trên sẽ trả kết quả là thời điểm trước khi bạn chạy my_function() và thời điểm sau khi hàm này đã kết thúc. Qua đó, bạn có thể nhìn thấy hàm my_function() chạy mất bao lâu.

Bạn có thể thử nó trên máy của mình, và tôi tin bạn có thể hiểu được cách chúng hoạt động.

Decorator cho hàm có tham số

Dưới đây là một ví dụ khác. Hàm print_number của chúng ta cần tham số được truyền vào. Rất đơn giản, thêm tham số cho hàm wrapper bên trong decorator là hàm của chúng ta có thể gọi tham số này từ trong decorator.

from time import sleep

def sleep_decorator(function):
    """Limits how fast the function is called."""

    def wrapper(*args, **kwargs):
        sleep(2)
        return function(*args, **kwargs)
    return wrapper

@sleep_decorator
def square_number(num):
    return num ** 2

print(square_number(222))

for x in range(1,6):
    print(square_number(x))

Decorator trên dùng để giới hạn tốc độ của hàm. Sau 2 giây, hàm mới được gọi 1 lần. Bạn có thể thử nó.

Sử dụng functools.wraps

Dưới đây là một ví dụ mà decorator được sử dụng nhiều nhất, đó là decorator login_required(). Decorator này để chắc chắn rằng người dùng đã đăng nhập và hệ thống và có quyền truy cập tới route được bảo mật (/secret).

from functools import wraps
from flask import g, request, redirect, url_for

def login_required(f):
    @wraps(f)
    def decorated_function(*args, **kwargs):
        if g.user is None:
            return redirect(url_for('login', next=request.url))
        return f(*args, **kwargs)
    return decorated_function

@app.route('/secret')
@login_required
def secret():
    pass

Trong ví dụ trên tôi đã sử dụng decorator functools.wraps(). Decorator này đơn giản là bảo vệ metadata của hàm được đóng gói.

Decorator có chứa tham số

Dưới đây là một ví dụ phức tạp hơn một chút. Chúng ta cần truyền tham số vào hàm decorator. Lấy ví dụ sleep_decorator ở trên, bây giờ chúng ta muốn truyền tham số là thời gian sleep. Để làm được việc này, trong lòng decorator chúng ta cần thêm 2 hàm lồng nhau nữa.

from time import sleep

def sleep_decorator(argument):
    """Limits how fast the function is called."""

    def real_decorator(function):
        def wrapper(*args, **kwargs):
            sleep(argument)
            return function(*args, **kwargs)
        return wrapper
    return real_decorator

@sleep_decorator(2)
def square_number(num):
    return num ** 2

print(print_number(222))

@sleep_decorator(3)
def cube_number(num):
    return num ** 3

for x in range(1,6):
    print(cube_number(x))

Kết luận

Trên đây là những hiểu biết của tôi về decorator được dùng trong Python. Decorator là một công cụ rất hữu dụng và được sử dụng khá thường xuyên. Hy vọng bài viết cho bạn phần nào hiểu biết về decorator, cách viết và sử dụng chúng.
